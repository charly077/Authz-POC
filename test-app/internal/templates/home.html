Why this project?

Most applications embed authorization logic directly in code — if user.role == "admin" scattered everywhere. This is fragile, hard to audit, and impossible to manage at scale.

This POC explores a better approach: externalized authorization. The application never makes access decisions itself. Instead, a dedicated infrastructure layer handles it, combining two complementary models:

📜
OPA — Policy-Based (ABAC)
Open Policy Agent evaluates attributes: request path, HTTP method, JWT claims, time of day, IP ranges. Policies are written in Rego and enforced at the gateway level by Envoy.
🔗
OpenFGA — Relationship-Based (ReBAC)
OpenFGA stores relationships between users and resources: “Alice is owner of Dossier”, “Bob is guardian of Alice”. Access is derived from the relationship graph, enabling fine-grained sharing.

Together, OPA handles the “can this user access this endpoint?” question, while OpenFGA answers “can this user see or edit this specific resource?”

Architecture
1Envoy Proxy intercepts every request
→
2Keycloak authenticates the user (OIDC)
→
3OPA evaluates the policy (allow/deny)
→
4App queries OpenFGA for resource-level access

The application code is completely free of authorization logic. It receives pre-validated identity headers from OPA and delegates relationship checks to OpenFGA.

How to test
  1. Sign in — Go to /home and log in as alice. You will be redirected to Keycloak, then back to the app with your identity verified.
  2. Create dossiers — On the /dossiers page, create a few dossiers (tax, health, or general). You are automatically the owner (stored as an OpenFGA relationship).
  3. Set up guardianship — Send a guardianship request to bob. Then sign in as bob in another browser to accept it. Bob becomes your guardian and can automatically see your dossiers.
  4. Grant a mandate — Back as alice, grant bob a mandate on a specific dossier. This writes a mandate_holder relationship tuple to OpenFGA, granting both view and edit access.
  5. Verify as Bob — Sign in as bob and check the Shared With Me section. Bob can now see (and edit) Alice's dossier — determined entirely by the OpenFGA relationship graph.
  6. Explore policies — Use the AI Manager to inspect, modify, or generate OPA policies using natural language.
